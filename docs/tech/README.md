# Tech Workspace

This directory stores technical implementation material and system diagrams.

## What goes here

- `implementation/`: technical implementation specs tied to approved PM artifacts.
- `diagrams/`: architecture and system diagrams.
- `templates/`: reusable technical doc templates.

## Relationship with PM workflow

- PM artifacts live in `docs/pm`.
- Technical implementation starts only after PRD, Review, and Tests stages are complete.
- Implementation artifacts here should always reference the originating PRD ID.
